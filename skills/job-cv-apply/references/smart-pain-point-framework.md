# SMART痛点逆向工程框架

> 从JD反推公司痛点，用SMART方案证明你能解决问题。
> 核心转变：不是"我要这份工作"，而是"这份工作需要我"。

---

## 方法论

### Step 1: 痛点逆推（JD → 公司真实痛点）

JD不会直接写"我们有这个问题"，但会通过要求暗示痛点：

| JD关键词/反复强调 | 暗示的真实痛点 |
|------------------|-------------|
| "cross-functional collaboration" | 跨部门协作效率低，部门墙严重 |
| "drive AI/digital adoption" | 技术落地推不动，业务部门抵触 |
| "P&L management" / "profitability" | 业务亏损，需要扭亏 |
| "build from scratch" / "0 to 1" | 新业务/新团队，没有基础 |
| "stakeholder management" | 上下游关系复杂，沟通成本高 |
| "process optimization" / "SOP" | 现有流程混乱，效率低 |
| "scale the team" / "grow the team" | 团队需要扩张，但缺管理经验 |
| "data-driven decision" | 决策靠拍脑袋，缺数据支撑 |
| "transformation" / "change management" | 组织变革阻力大 |
| "fast-paced environment" | 业务变化快，团队跟不上节奏 |

### Step 2: SMART方案构建

每个痛点对应一个SMART方案：

```
痛点: [从JD逆推的公司真实痛点]

S - Specific: 具体要做什么
    ✗ "提升团队效率"
    ✓ "建立华东区SOP推行体系，覆盖门店运营+客户服务+库存管理三个核心流程"

M - Measurable: 可量化指标
    ✗ "显著提升"
    ✓ "团队交付周期缩短30%，客户投诉率下降50%"

A - Achievable: 你的经验证明（引用STAR故事银行中的真实案例）
    ✗ "我相信我能做好"
    ✓ "在XX公司，我用同样方法将650人团队利润从-15%提升到+8%（见STAR故事1）"

R - Relevant: 如何对齐JD核心诉求
    ✓ "这与JD中要求的'process optimization'和'regional coordination'直接对应"

T - Time-bound: 明确时间线
    ✗ "尽快完成"
    ✓ "入职30天完成诊断，60天完成方案设计，90天推行第一轮"
```

### Step 3: 面试话术（30秒版本）

每个SMART方案浓缩为30秒可讲完的话术，中英文各一版：

**中文模板：**
```
"我在[公司]遇到过类似的情况。当时[痛点描述]，我用[S-具体做法]，
在[T-时间]内实现了[M-量化结果]。对这个岗位，我计划[简述方案]，
第一个月先[诊断动作]，三个月内[首轮成果]。"
```

**英文模板：**
```
"I faced a similar situation at [company]. The challenge was [pain point].
I implemented [S-action] and within [T-timeframe], achieved [M-result].
For this role, I'd start with [diagnosis action] in month one,
and deliver [first milestone] within 90 days."
```

## 常见痛点模式 × SMART方案库

### 模式1: AI落地推不动

**痛点**: 技术团队做了AI产品，业务部门不愿用
**SMART方案**:
- S: 选一个业务部门做AI试点，跑通"选型→培训→SOP嵌入→效果量化"全流程
- M: 试点部门AI工具使用率>70%，效率提升>20%
- A: STAR故事3（SOP数字化，15000+反馈处理）
- R: 对齐"drive AI adoption"要求
- T: 30天选部门+选工具，60天培训+试运行，90天出效果报告

### 模式2: 业务亏损需扭亏

**痛点**: 某业务线/区域持续亏损
**SMART方案**:
- S: 驻场诊断→止血→优化→增长四步走
- M: 3个月内止损，6个月内利润转正
- A: STAR故事1（华东区-15%→+8%）+ 故事5（门店扭亏）
- R: 对齐"P&L management"要求
- T: 30天诊断，60天止血方案落地，90天验证初步效果

### 模式3: 跨部门协作低效

**痛点**: 各部门各干各的，项目推进慢
**SMART方案**:
- S: 建立跨部门协作机制（例会+问题跟踪+联合KPI）
- M: 项目交付周期缩短25%，跨部门问题响应时间<24h
- A: STAR故事4（华东+华南跨区域协调）
- R: 对齐"cross-functional"和"stakeholder management"
- T: 30天摸清协作痛点，60天建机制，90天跑通第一个跨部门项目

### 模式4: 新业务/新团队从零开始

**痛点**: 新市场/新产品线，没有基础
**SMART方案**:
- S: 市场调研→团队搭建→渠道建设→首单突破
- M: 6个月内签约10家核心客户/渠道
- A: STAR故事2（从0到1建立经销商网络）
- R: 对齐"build from scratch"和"0 to 1"
- T: 30天市场调研，60天团队+渠道框架，90天首单

## 90天入职规划模板

### Day 1-30: 诊断期
**目标**: 了解全貌，建立信任
**具体动作**:
- 与直属上级1v1：明确期望、KPI、优先级
- 与核心团队成员1v1（每人30分钟）：了解痛点、能力、士气
- 与关键stakeholder 1v1：了解上下游需求和摩擦点
- 梳理现有流程/SOP/文档
- 输出诊断报告

**交付物**: 诊断报告（现状+问题清单+优先级排序）

### Day 31-60: 方案期
**目标**: 制定方案，争取试点
**具体动作**:
- 基于诊断结果，制定改进方案（SMART格式）
- 与上级对齐方案，争取资源和授权
- 选一个小范围试点（一个团队/一个项目/一个区域）
- 建立KPI基线（记录改善前的数据）

**交付物**: 改进方案+试点计划+KPI基线

### Day 61-90: 落地期
**目标**: 推行第一轮改进，量化初步成果
**具体动作**:
- 推行试点
- 跟踪KPI变化
- 收集反馈，调整方案
- 向上级汇报首轮成果
- 规划下一轮扩展

**交付物**: 首轮成果报告+长期路线图

**⚠️ 关键原则**:
- 90天规划必须结合SMART痛点方案，不能泛泛而谈
- 每个阶段的交付物必须可量化
- "诊断期"不是"学习期"——你是在用专业眼光审视问题，不是来学习的

## 使用流程

```
面试准备触发
  ↓
Step 0 (场景14): 读取JD → 逆推2-3个痛点
  ↓
从本文件选择匹配的痛点模式 → 定制SMART方案
  ↓
从STAR故事银行(star-stories.md)选取匹配故事 → 填入A字段
  ↓
生成30秒话术(中英文)
  ↓
基于痛点方案生成90天规划
  ↓
输出面试准备卡(场景14 Step 5)
  ↓
可选: 生成面试PPT(场景16)
```
