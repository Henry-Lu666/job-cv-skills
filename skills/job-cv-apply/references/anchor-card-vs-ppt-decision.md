# 面试准备：PPT vs 锚点卡决策指南

> 2026-07-07 Honwell实战验证 — 不是所有面试都要做PPT

---

## 决策树

```
Stage 6A公司调研完成
       ↓
判断公司类型+面试场景
       ↓
┌──────────────────────────────────────────────┐
│  大公司结构化面试                               │
│  (HR+业务负责人+VP逐轮/需投影/竞标式)           │
│  → 做PPT（7页，每页≤50字）                     │
└──────────────────────────────────────────────┘
       OR
┌──────────────────────────────────────────────┐
│  中小企业/Director/Owner直接面                │
│  咖啡面/非正式会面                              │
│  实干型老板（不喜欢听概念）                     │
│  → 做锚点卡（1页A4纸）                         │
└──────────────────────────────────────────────┘
       OR
┌──────────────────────────────────────────────┐
│  不确定                                        │
│  → 先做锚点卡快速准备                          │
│  → 问用户："这种场景我觉得锚点卡更实用，         │
│     你看要不要PPT？"                           │
└──────────────────────────────────────────────┘
```

## 锚点卡模板（参考Honwell实战）

### 结构

| 板块 | 内容 | 长度 |
|------|------|------|
| 开门第一句 | 人话起手式，带画面感的一句话自我介绍 | 1句 |
| 三个核心信息 | 你最想让面试官记住的三件事 | 各1句 |
| SMART痛点(压缩) | 2-3个痛点×方案×一句话证据 | 各3-4行 |
| 可能被刁难的问题 | 预判5-6个攻击点×应答策略 | 各2-3行 |
| 你要问他的问题 | 2-3个高质量反问 | 各1-2行 |
| 红线 | ❌不说什么，✅一定要说 | 各3-5条 |

### Honwell实战经验

**为什么锚点卡比PPT更合适：**
- 中小企业面试通常是面对面聊天，不是投影演示
- 老板/owner直接面，他们看"这人靠不靠谱"多于"方案是否完美"
- PPT在中小企业可能传递负面信号：太正式/太流程化/不接地气
- 一张A4纸能讲清楚的事，做成7页PPT反而显得心虚
- 实操结果：SMART方案文档(3页) + 锚点卡(1页) 足够应对1小时深度对话

**Honwell案例关键洞察：**
- 公司定位：21年家具商转型医疗科技，需要"帮藤椅厂装成工程科技公司去投标"
- 面试官类型：实干型老板（务实、直接、不爱听概念）
- 用户定位：不是"AI专家"而是"技术翻译官+投标军师"
- 三个核心信息：翻译官角色 / 懂政府大客户逻辑 / 诚实但有方法论

## PPT模板（参考）

当需要做PPT时，7页结构：

| 页码 | 内容 |
|------|------|
| 1 | 封面（公司名+岗位+你的名字+日期） |
| 2 | 你对公司的理解（用3句话证明你真的调研过） |
| 3-5 | SMART痛点方案（每页1个痛点） |
| 6 | "你可能会担心"（主动化解大公司病/行业经验等顾虑） |
| 7 | 90天规划（时间线+里程碑+交付物） |

**规律：** PPT每页≤50字关键词，话术用嘴讲。备注栏放完整演讲稿。
