# 外部内容评估框架

> 当用户分享外部内容（视频、文章、工具推荐）并问"有没有用"/"能不能进知识库"/"帮我整理"时，用此框架评估。

## 评估三步法

### Step 1: 真实性检查
- 引用的工具/概念是否存在？（搜不到=可能AI幻觉）
- 链接是否有效？（404=无法验证原始内容）
- 是一手技术文档还是二手AI解读？（二手解读可信度低）

### Step 2: 对标现有体系
- 用户已有哪些能力覆盖了这个内容？
- 是否有更强的现有方案？（如：Hermes cron > 手动自动化，last30days > 付费热点API）
- 增量价值是什么？（必须明确说出"这个能做但你现有的做不到X"）

### Step 3: 风险/成本评估
- 合规风险？（爬虫API、数据版权）
- 额外成本？（付费API、外部模型 — 用户红线）
- 维护成本？（多一个工具=多一个出bug的点）

## 输出格式

```
结论: 采纳 / 部分采纳 / 不采纳
理由: 一句话
增量价值: (如果有) 具体说明哪个能力是新的
风险: (如果有) 具体说明
建议动作: 进知识库 / 融入现有skill / 仅参考 / 跳过
```

## 已验证案例

| 内容 | 评估结果 | 核心理由 |
|------|---------|---------|
| Hermes刹车+导航(SkillO) | ❌ 不采纳 | SkillO不存在(AI幻觉)，链接404，Hermes原生已有这些能力 |
| SMART面试法(TRAE Work) | ✅ 部分采纳 | SMART框架有实操价值，融入job-cv-apply skill；TRAE Work本身是字节AI IDE，不需额外工具 |
| RedFox红狐数据API | ❌ 不采纳 | 本质是付费爬虫API，合规风险高，已有last30days-cn免费覆盖热点需求 |
| Obsidian智能工作台 | ❌ 不采纳 | 用户已在L6-L7阶段(Agent级)，Dashboard是L3方案，体感无提升 |

## Pitfall

- **不要因为"内容质量高"就推荐采纳** — 质量高≠对用户有用。评估标准是增量价值，不是内容本身好坏
- **不要主动帮用户"联网搜索补充"** — 除非用户明确要求。用户说"看看有没有用"是评估请求，不是研究请求
- **用户说"没必要了"时立即停止** — 不要追问"要不要存个档"或"以后可能有用"
