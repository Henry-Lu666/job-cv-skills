# SMART方案用于Networking Meeting（无JD场景）

## When to Use

当用户面对的不是正式面试（有JD的岗位），而是与创业公司创始人的networking meeting时，SMART痛点方案需要从"JD逆推痛点"改为"调研反推痛点"。

## 与标准SMART流程的差异

| 标准面试（有JD） | Networking Meeting（无JD） |
|-----------------|--------------------------|
| 痛点从JD逆推 | 痛点从公司调研反推 |
| 胜任力矩阵对齐JD要求 | 胜任力矩阵对齐公司业务需求 |
| 90天规划围绕岗位职责 | 90天规划围绕公司发展阶段 |
| PPT 7页标准模板 | PPT可扩展到8-9页，信息不密集 |
| 话术强调"我能胜任这个岗位" | 话术强调"我能帮你解决这些问题" |

## 痛点反推方法

1. 做深度公司调研（用deep-research skill的公司调研模式）
2. 从调研中找3类痛点：
   - **运营痛点**：产能瓶颈、流程混乱、团队缺口
   - **战略痛点**：新业务线需要架构、市场扩展需要资源
   - **客户痛点**：目标客户有文化/语言/决策链的gap
3. 每个痛点挂载你的简历真实经历（不是JD要求，是你能提供的）
4. 用SMART结构包装，但A（Achievable）部分更强调"迁移经验"而非"直接匹配"

## PPT结构建议（创始人meeting）

```
Page 1: 封面（定位句，不是"应聘XX岗位"）
Page 2: 业务洞察（证明你做了深度调研）
Page 3-N: 痛点方案（每页2-3点，不密集）
Page N+1: 资源盘点（韦恩图/能力矩阵）
Page N+2: 90天承诺（可验证的交付物）
Page N+3: 动机故事（为什么是这家公司）
Page N+4: 问题清单（把舞台交给对方）
```

## 关键话术模式

**创始人最想听的不是"我有多厉害"，而是"你怎么帮我"。**

- ✅ "你在前面跑业务，我在后面把家看好"
- ✅ "把你从日常运营中解放出来"
- ❌ "我有12年管理经验"（太自夸）
- ❌ "你的团队可能搞不定XX"（攻击性）

## "大公司病"顾虑化解

当你的背景是大公司/国企，对方是创业公司时，必须主动化解"大公司病"顾虑：
- 用"大公司内创业"经历对冲（在大公司里做过从0到1的事）
- 用"短周期交付+快速复盘"经历证明适应力
- 用"一人就是一支团队"的经历证明能独立作战

## 视觉设计：用目标公司品牌色

当面试对象是品牌敏感型创始人时，PPT用目标公司的品牌色而不是个人品牌色：
- 潜台词："我在视觉上已经准备好融入团队了"
- 操作：从目标公司官网提取CSS颜色（browser_console提取getComputedStyle）
- 生成后微调：改背景色+主色+强调色
