# 求职核心规则

> 所有求职操作的基准参照。本skill定义了求职全流程的**红线、铁律、甜蜜区定位和决策框架**。

---

## 前置红线

### 文件存储规则

所有求职相关文件（调研报告、JD分析、面试准备、PPT大纲等）统一放 `08-找工作/` 下的对应公司目录。

| ❌ 错误做法 | ✅ 正确做法 |
|-----------|-----------|
| 调研存入 `concepts/` 作为通用知识 | 存入 `08-找工作/{公司名}/`，因为是求职场景专属 |

### 简历操作铁律

- **定制简历基于V4 base**：简洁bullet风格（关键词：内容，40-70字/条），无子标题，中英双语
- **英文bullet风格**：与中文一致——无主语短句，keyword: content格式，不用完整句子
- **只出docx，不转PDF**（用户手动转PDF）
- **定制前必须验证岗位存活**：打开JD页确认不是404/已关闭——这是血泪教训
- **简历内容幻觉红线**：任何百分比/评级/排名类数据必须确认有实测支撑。宁可不写数字，不编造
- **python-docx编辑**：插入新bullet必须从同类型bullet段落deepcopy，不能从heading/title复制

### 追踪表规则

追踪表是Markdown文件，按状态分组（等反馈/无反馈/被拒/枪毙），所有条目带JD链接。
- JD文件命名：`{ID}_{公司简称}_{岗位简称}.md`
- 每个JD文件底部必须有`## 中文翻译`章节
- P1/P2所有岗位都必须保存JD文件到JD库

### 已登录tab优先

CDP模式下先检查已有tab，用CDP Runtime.evaluate直接操作登录tab。不擅自开新tab。
- 对任何岗位/邮件/建议做出分析**之前**，必须先读取个人简历
- **Stage 1铁律：简历定制前先验证岗位是否存活**

## 甜蜜区定位

| 方向 | 匹配度 | 说明 |
|------|--------|------|
| **BD / 战略合作 / 客户管理** | 主力方向 | 大客户开发+渠道管理+GTM经验直接匹配，BD岗对行业经验容忍度最高 |
| **AI落地推行 / 变革管理 / 业务赋能** | 主力方向 | SOP推行+区域协调+推动业务部门采用新技术=AI Adoption岗核心要求 |
| **数字化转型 / 流程优化** | 主力方向 | SOP标准化+流程再造+跨区域协调+变革管理 |
| AI应用开发工程师 | 副线 | RAG/Agent/Prompt Engineering + AI Native开发，行业壁垒低 |
| AI方案销售/BD（技术型） | 副线 | AI背景+BD能力差异化 |
| 低门槛AI PM | 观望 | 仅限要求"AI exposure"而非"3-5yr PM experience"的岗位 |
| Head of AI / AI Director | 暂停 | 要求10-15年AI领导经验 |

**核实岗位时3个主力方向都必须覆盖**，不要只盯一个方向。

## 决策分析框架：双视角法

当需要决策「是否投递某岗位/是否参与某机会」时，采用**实践者+经济学家双视角分析法**。

### 实践者视角（落地执行）

1. **投递时机** — ATS按提交时间排序；周日晚上投递=周一早HR打开时排最前
2. **HR侧信号** — 同一公司投多个岗位时，ATS会关联显示。主动在Cover Letter说明选择原因
3. **硬性障碍** — 逐一核实签证、语言要求、行业经验
4. **差异化竞争** — 非传统背景（如非咨询进入Strategy）可转化为"执行可信度"优势
5. **组织现实** — 机构性质影响招聘偏好：非营利≠纯商业公司，对多样性容忍度更高

### 经济学家视角（资源配置）

1. **机会成本** — 投递时间（~2分钟）+ 定制精力 vs 期望年薪
2. **预期价值** — 薪资中位值 × 面试概率的估算
3. **沉没成本** — 已花的时间是沉没成本，决策时不应考虑
4. **期权价值** — 履历背书、人脉资源、行业切换通道的品牌光环
5. **边际成本** — 从定制版改投下一个岗位的边际成本远低于首次定制

### 输出格式
```
判断：✅ 投 / ❌ 不投 / ⚠️ 有条件投（列出条件）
───
实践者视角：
- 核心发现（证据强度：高/中高/中）
经济学家视角：
- 机会成本：XXX
- 预期价值：XXX
- 期权价值：XXX
```

## 岗位优先级：渠道权重 ≥ 匹配度

纯靠匹配度评分决定押哪匹马是错误的。**内推渠道的权重应视为冷投的1.5-2x。**

| 因素 | 内推 | 海投 |
|------|------|------|
| 简历被HR完整阅读 | ✅ 推荐人背书→至少看完 | ❌ ATS筛+HR扫3-5秒 |
| 简历→面试转化率 | ~30-50% | ~2-5% |
| 推荐人信息价值 | ✅ 可提前了解团队 | ❌ 无 |

**正确评估顺序：** ①确认有无内推/渠道优势 → ②看匹配度评分 → ③综合判断优先级。

## SME行业推断陷阱

不要从产品线/经营范围自行推断SME的行业属性。必须查JD全文+客户类型。

**正确做法：**
- 读JD全文确认实际工作场景（招投标 vs 现场施工 vs 产品研发）
- 查客户类型决定实际工作语言和协作模式
- 对SME特别注意「一职多岗」——JD写的5条职责在实际工作中可能是10条

## JD语言信号解读：「advantageous/preferred」的真实权重

当JD中写语言/资质要求为"advantageous""preferred but not required"时，**不能只看字面折扣**。必须结合公司实际客户结构和业务布局判断真实需求强度。

**正确做法：**
1. 看到"advantageous/preferred/plus"等措辞，先搜索公司的客户分布和业务区域
2. 如果客户主体在/含中国→普通话权重应显著上调
3. BD/销售/客户管理类岗位的语言要求权重>后台/技术类岗位
4. 在追踪表评估时标注「JD字面写X，但业务实际需要Y」
