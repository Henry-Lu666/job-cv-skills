# 求职面试准备

> 核心理念：不是"我要这份工作"，而是"这份工作需要我"——从JD反推公司痛点，用SMART方案证明你能解决。

---

## 面试准备流程

### Step 0：判断会面类型

| 类型 | 策略 |
|------|------|
| **正式面试**（有HR/有JD/在办公室） | 标准的SMART痛点逆向工程 |
| **非正式咖啡会面**（咖啡厅/Networking/创始人面） | 画像+锚点卡流程 |
| **关键区别** | 正式面试展示能力→匹配需求；咖啡会面让对面放心"这人靠谱" |

### Step 1：公司深度调研

进入正式准备前，先做公司调研。**不要只看官网**，多源交叉验证：
- 公司客户结构（决定实际工作语言）
- 技术/IP声明（用专利→招聘→展会5层框架验证虚实）
- 竞品/行业位置
- 近30天新闻/舆情

### Step 2：SMART痛点逆向工程（面试核心武器）

从JD反推公司痛点，构建SMART方案。每个痛点输出面试话术（中英文各一版，30秒可讲完）。

```
═══ SMART痛点方案 — {公司} {岗位} ═══

痛点1: [从JD逆推的公司真实痛点]
  SMART方案:
    S: [具体做什么]
    M: [可量化指标]
    A: [你的经验证明]
    R: [如何对齐JD要求]
    T: [时间线]
  面试话术(30秒): "..."
```

**痛点逆推技巧：**
- JD反复强调"cross-functional" → 痛点：跨部门协作效率低 → 你的方案：跨区域协调经验
- JD要求"AI adoption" → 痛点：技术落地推不动 → 你的方案：变革管理+SOP推行方法论
- JD强调"P&L management" → 痛点：业务亏损需扭亏 → 你的方案：扭亏实战

### Step 3：STAR故事构建

对JD中的每条核心要求，从简历经历中提取STAR故事：

| JD要求 | S情境 | T任务 | A行动 | R结果 | 反思 |
|--------|-------|-------|-------|-------|------|
| 跨区域团队管理 | 某区域团队管理场景 | 扭亏为盈 | 推行SOP+区域协调 | 利润显著改善 | 关键先稳人心再改流程 |

### Step 4：高频问题准备

| 问题类型 | 常见问题 | 准备要点 |
|----------|---------|---------|
| 自我介绍 | "Tell me about yourself" | 30秒电梯演讲，突出跨界组合 |
| 为什么转型 | "Why the change?" | 从运营数据化→AI落地→硕士深造 |
| 职业规划 | "Where do you see yourself in 3 years?" | 对齐目标岗位晋升路径 |
| 失败经历 | "Tell me about a failure" | 坦诚困难→方法论→结果 |
| AI落地 | "How do you drive AI adoption?" | SOP推行+变革管理经验 |
| 薪资期望 | "What's your salary expectation?" | 见薪资谈判章节 |

### Step 5：入职90天规划

| 阶段 | 目标 | 具体动作 | 交付物 |
|------|------|---------|--------|
| Day 1-30 | 诊断期 | 了解团队/业务/痛点，1v1关键人访谈 | 诊断报告 |
| Day 31-60 | 方案期 | 制定改进方案，小范围试点 | 改进方案+试点计划 |
| Day 61-90 | 落地期 | 推行第一轮改进，量化初步成果 | 首轮成果报告+路线图 |

⚠️ 必须结合SMART痛点方案，不能泛泛而谈。

### Step 6：反问面试官的问题

1. 这个岗位的KPI是什么？
2. 团队目前最大的挑战是什么？
3. 上任做了什么？为什么离开？
4. AI/数字化在公司处于什么阶段？
5. 入职第一个月，你希望我优先解决什么问题？

## 咖啡会面策略

### 识别面试真实性质

| 信号 | 解读 |
|------|------|
| 约在咖啡厅而非办公室 | 可能无实体办公室/或为vibe check |
| Founders/Co-founders亲自来 | 对方亲自把关，信任度比HR面高 |
| 明确说"get to know you" | 非正式interview，是看人 |
| 聊了个人话题（孩子/家庭） | 已建立私人连接——正向信号 |

### 咖啡会面对话路径（30分钟）

```
0-5 min:  破冰—感谢邀约，自然过渡
5-10 min: 对方主导—让Founder先聊公司现状和痛点
10-20 min: 你的价值—基于对方痛点，讲述能解决的1-2个核心问题
20-25 min: 提问环节—准备3-5个有深度的问题
25-30 min: 收尾—了解下一步
```

**关键原则**：让对方主导前10分钟——创始人想聊他的公司，让他聊。你从他的倾诉中捕捉痛点，然后精准回应。

### 咖啡后跟进策略

| 时间 | 动作 |
|------|------|
| 当天下午 | 简短感谢信 |
| 2-3天后 | 轻触发送补充资料 |
| 1周后无反馈 | 礼貌follow-up |
| 2周后 | 视为低概率，继续其他机会 |

## 薪资谈判

### Step 1：了解谈判阶段

| 阶段 | 策略 |
|------|------|
| 初筛填写期望 | 给范围，不给具体数字 |
| HR电话问期望 | 反问预算（"Could you share the range first?"） |
| 终面后谈offer | 用市场数据+自身价值谈判 |

### Step 2：市场数据准备

参考来源（按优先级）：
- **Glassdoor** — 具体公司薪资数据
- **Michael Page Salary Guide** — 行业薪资范围
- **Morgan McKinley Salary Guide** — 行业薪资范围
- **JobsDB薪资中心** — 香港本地数据
- **Robert Half Salary Guide** — 交叉验证

⚠️ **薪资数据必须来自真实来源，禁止编造数字。**

### Step 3：谈判脚本

**场景A: HR问期望薪资**
```
HR: "What's your salary expectation?"
你: "I'm flexible. Could you share the budget range for this role?"
HR: [给范围]
你: "That's within my expectations. I'm excited about [公司] because [理由]."
```

**场景B: Offer低于期望**
```
你: "Thank you for the offer. Based on my research and this role's scope,
I was expecting closer to [目标数字]. Could we discuss if there's room?"
```

**场景C: 用竞争Offer谈判**
```
你: "I want to be transparent—I have another offer at [范围].
However, [这家公司] is my top choice. If we can close the gap, I'd accept today."
```

### Step 4：谈判优先级

| 谈判项 | 优先级 | 说明 |
|--------|--------|------|
| Base Salary | ★★★ | 基本工资，影响最大 |
| Bonus | ★★☆ | 年终奖/绩效奖 |
| 签证支持 | ★★★ | 对非本地身份者是关键 |
| 年假 | ★☆☆ | 香港法定7天，好公司15-20天 |
| 远程/弹性 | ★★☆ | 值得谈，尤其跨区域岗位 |

## 面试PPT生成

### PPT结构（7页标准模板）

| 页码 | 内容 |
|------|------|
| 1 | 封面—姓名+目标岗位+一句话定位 |
| 2 | 痛点诊断—从JD逆推的2-3个公司痛点 |
| 3-4 | SMART解决方案（核心页，每个痛点一页） |
| 5 | 胜任力矩阵—JD要求→你的经验→证据数据 |
| 6 | 入职90天规划 |
| 7 | 总结+三个关键词+联系方式 |

### 每页附带演讲稿

每页PPT输出30-60秒演讲稿（中文版）：
- Page 1: 30秒自我介绍
- Page 2: 60秒痛点诊断
- Page 3-4: 各60秒方案讲解
- Page 5: 60秒胜任力说明
- Page 6: 60秒90天规划
- Page 7: 30秒收尾

**内容密度偏好**：不限7页，可扩展到8-9页。每页2-3点，不超过3点。PPT只放关键词，话术用嘴讲。

### 生成方式

1. 制作设计大纲（含配色、字体、布局）
2. 验证公司官网品牌色（提取CSS变量）
3. 用对应工具生成PPT
4. 双版本交付（暖色版+纯白降级版以适应不同投影环境）
