# 更新日志

## v2.0.0 (2026-07-13)

### 架构重构：单体→微技能套件

v2.0是求职skill体系的重大升级。**从1个单体技能（job-cv-apply，100KB）拆分为6个正交的独立模块**，每个专注一个职责维度，可独立使用也可组合。

| 维度 | v1.0（单体） | v2.0（微技能套件） |
|------|-------------|-------------------|
| 组织结构 | 1个超大skill：job-cv-apply（100KB） | 6个独立模块：job-core / job-pipeline / job-cdp / job-interview / jd-inspector / interview-simulation |
| 设计原则 | 所有逻辑塞一个文件 | 职责正交、引用共享、单文件≤10KB |
| 触发效率 | 加载整个100KB即使只用一小部分 | 按需加载，每段独立触发词 |
| 维护成本 | 改一处影响全局 | 模块化，patch不影响其他模块 |

### 全链路整合：调研→SMART→PPT→模拟

v1.0中调研、SMART、PPT、模拟4个环节松耦合，用户需要手动串联。v2.0通过跨skill编排形成完整前置准备流水线：

```
公司深度调研（deep-research多Agent调研）
  → 5角色对抗验证（STORM框架）
  → SMART痛点逆向工程（从JD反推公司痛点×你的经验）
  → 多AI交叉迭代（至少2轮独立AI对照）
  → PPT设计大纲 → ppt-master生成
  → 多轮面试模拟（interview-simulation）
  → 修正锚点卡 + 画句号清单
```

其中ppt-master支持自动提取公司官网品牌色、双版本交付（暖色+降级白）、演讲稿放备注栏。调研支持多源交叉验证（专利→招聘→展会→产品→面试 5层框架）。

### 重大方法论升级

#### 1. STORM 5角色对抗分析框架（v2.0核心升级）

v1.0只有双视角（实践者+经济学家）。v2.0引入完整的**5角色对抗框架**，覆盖从JD分析→调研→面试叙事审计的全流程。

| 角色 | 视角 | 作用 |
|------|------|------|
| 学者 | 理论/方法论/研究数据 | 判断框架合理性，找行业基准 |
| 历史学家 | 演进/归因/先例 | 追溯问题根因，识别重复模式 |
| 实践者 | 落地/约束/可执行 | 真实世界怎么干，HR和ATS怎么想 |
| 经济学家 | 资源配置/机会成本 | 算期望值，对抗沉没成本谬误 |
| 怀疑者 | 挑刺/反方/极限测试 | 专攻论证链薄弱环节，避免盲区 |

**分组对抗（3组并行）：** 学者+历史学家（理论组）→ 实践者+经济学家（实证组）→ 怀疑者单独（压力组）→ 观点对撞 → 输出修正清单。适用场景：文档质量审查、面试叙事架构审计、公司调研结论验证、多AI交叉验证。

#### 2. 行业壁垒判定：从二分类到刻度变量（v1.0→v2.0核心升级）

**v1.0：** 看到行业名直接排除（"航空/赛马×"），错失大量可迁移的机会。

**v2.0：** 区分核心业务岗（需行业资质）vs 后台赋能岗（方法论可迁移）。

```
v1.0: 看见"赛马"→❌排除
v2.0: 读JD前3条→确认是绩效管理赋能岗→🟢继续评估
```

#### 2. 双视角决策框架（v2.0新增）

**实践者视角**（HR/SOP层面的落地执行洞察）+ **经济学家视角**（机会成本/预期价值/期权价值的资源理性评估），双视角交叉验证，避免单一视角偏差。

#### 3. 渠道杠杆原则（v2.0新增）

**内推权重 ≥ 匹配度分数。** 30%匹配度的内推岗面试转化率可能高于80%的海投岗。评估公式：综合权重 = 渠道确定性 × 匹配度 × 面试转化率。

#### 4. 手机阅读适配标准（v2.0新增）

所有cron输出统一采用25汉字/行、粗体分段、无序列表、无表格格式——适应飞书/即时通讯平台的手机端阅读体验。

#### 5. 弹性cron调度（v2.0新增）

输出型daily cron从固定时间（`0 9 * * *`）改为弹性间隔（`every 24h`），避免agent断线时连续错过执行。

### 流程完善

- **同公司多岗位投递间隔建议**：不同部门≥3天，同部门≥7天
- **"快速通道"场景**：日常巡检时跳过重新评估，仅做存活验证，全流程降至5-10分钟
- **JD语言信号解码**："advantageous/preferred"的权重必须结合公司业务布局判断
- **"Chinese"语境歧义**：香港本地机构写"Chinese"通常指粤语，外企/MNC可能接受普通话
